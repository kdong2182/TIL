# Github로 협업하기
## .gitignore
### 1. .gitingnore 이란?
> 특정 파일 혹은 폴더에 대해 Git이 버전 관리를 하지 못하도록 지정하는 것

- .gitignore를 하는 이유
  - 불필요한 파일이 커밋되는 것을 막기 위해
  - 개인 환경 차이로 인한 충돌을 줄이기 위해
  - 민감정보가 실수로 올라가는 걸 줄이기 위해
  - 저장소를 가볍고 재현 가능하게 하기 위해


## clone , pull
### 1. clone
> 원격저장소 자체를 복제

`git clone <원격 저장소 주소> <복제한 레포지토리 폴더이름 지정>`의 형태로 작성합니다.

처음에 한 번만 하면 됩니다


### 2. pull
> 원격저장소의 변경사항을 가져와서, 로컬저장소를 업데이트

`git pull origin main` 업데이트된 내용을 가져옵니다.

### 3. push
> 업데이트된 내용을 다시 업로드 하기 위해 사용합니다.

`git push origin main` 

- 참고
origin은 일종의 도로명이라고 생각하면 됩니다.
도로명을 origin이라고 하는 것은 일종의 관례입니다.

### 4. Branch
> `Branch`는 작업공간을 나누어 독립적으로 작업하도록 돕는 도구입니다.

- Branch를 사용하는 이유
  - 항상 공개되어 협업에 용이
  - main은 공용공간이고, branch는 개인공간
  - 고객들이 사용중인 경우 방해하지 않음


1. Branch
- git branch
  > 브랜치 조회, 생성, 삭제 등 브랜치와 관련된 Git 명령어
  ```bash 
  # 브랜치 목록 확인
  $ git branch

  # 원격 저장소의 브랜치 목록 확인
  $ git branch -r

  # 새로운 브랜치 생성
  $ git branch <브랜치 이름>

  # 특정 커밋 기준으로 브랜치 생성
  $ git branch <브랜치 이름> <커밋 ID>

  # 특정 브랜치 삭제
  $ git branch -d <브랜치 이름> # 병합된 브랜치만 삭제 가능
  $ git branch -D <브랜치 이름> # (주의) 강제 삭제 (병합되지 않은 브랜치도 삭제 가능) 
  ```
  
- git switch
  > 현재 브랜치에서 다른 브랜치로 `HEAD`를 이동시키는 명령어
  `HEAD`란 현재 브랜치를 가리키는 포인터를 의미합니다.


- Vim 간단 사용법

